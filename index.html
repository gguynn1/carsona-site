How Data Reaches the Dashboard

Carsona fuses data from four independent sources running at different speeds, then normalizes everything into a single coherent snapshot before events or the AI ever see it.

OBD-II (via BLE) delivers the core engine data — RPM and speed arrive together at roughly 8 Hz. Secondary PIDs (coolant, intake temp, fuel status, MAF, and more) now use an adaptive scheduler, so driver-facing values like throttle, load, coolant, and fuel refresh faster while slower diagnostic channels continue rotating in the background without overwhelming the adapter.

Device sensors run at 20 Hz for G-force and jerk via CoreMotion, plus ~1 Hz for GPS position, speed, heading, and altitude. A phone-mount orientation setting transforms raw accelerometer axes into vehicle-frame lateral, longitudinal, and vertical G-forces — so the reading is correct regardless of how your phone is mounted.

Weather & location are fetched via Open-Meteo (temperature, humidity, wind, conditions) and Apple reverse geocoding (road names, city context). Reverse geocoding now refreshes on a shorter time gate, speed-adaptive movement thresholds, and meaningful heading changes so street-name context catches up faster after turns. In Road Trip mode, nearby landmarks are discovered through MapKit for richer AI commentary.

Because the event detector evaluates at ~8 Hz but motion sensors sample at 20 Hz, Carsona uses peak-hold tracking — the highest G-force magnitude between evaluations is captured and drained, so brief spikes (hard brake, pothole) are never lost between polls. Every reading is timestamped and checked for freshness before cross-source comparisons happen.

Two Commentary Paths

Your car doesn’t just react to events — it also speaks up during quiet stretches when it has something interesting to say.

Event-Driven Path

When something happens — hard braking, a launch, cornering, altitude change — nearby events are buffered into a “moment,” grouped, and processed as a single compound remark. Three events in quick succession become one observation, not three separate interruptions.

Idle Commentary Path

After a configurable quiet period with no remark spoken, the car draws from what it senses — its metrics, the weather, its location, and its own state — to find something interesting to say unprompted.

Both paths share the same prompt pipeline and speech output, so the car’s voice and personality remain consistent whether it’s reacting to a redline pull or musing about the sunset during a highway cruise.

Personality-First Prompt Architecture

Every prompt Carsona sends to the AI starts with who the car is, not what just happened. Character comes first, data comes second. This is the order the AI sees on every single remark.

  1. Character opening — the first-person personality description you chose (or a generic fallback).
  2. Voice baseline — core first-person vehicle identity, humor guidelines, and the rule that humor drops when a real safety warning matters.
  3. Profile traits — the personality traits you configured, framed as “Your character adds:” on top of the first-person baseline.
  4. Speech habits — core quirks plus any vehicle-specific quirks from the personality config (e.g. how the car references its own engine or heritage).
  5. Boundaries — safety rules, anti-distraction guardrails, and any custom boundaries you’ve set.
  6. Vehicle specs — the car’s own specs presented as self-knowledge, so the AI can reference its own horsepower, weight, or drivetrain naturally.
  7. Tone layering — additional tone guidance when configured (warm, sarcastic, technical, etc.).
  8. Driver name — woven into the greeting naturally, not bolted on.
  9. Few-shot examples — sample remarks from the personality config that show the AI exactly how this character sounds.
  10. Compact rules — brevity targets, anti-recitation, safety reminders, and shape-variety guidance to prevent repetitive structure.
  11. Mode overlay — additional rules for Road Trip, Chill, Night Drive, Spirited, or other assistant modes when active.

After the personality layers, the AI also receives the last 12 spoken remarks with relative timestamps, so it has rolling memory of what it already said. A shape-variety detector checks the last 3 remarks for structural repetition and nudges the AI toward different openings when patterns emerge. Older spoken lines are compressed into a session-arc summary that preserves trigger context (event vs idle vs conversation), drive phase, road name, and event labels where available.

AI System & Event Engine

Events are detected, grouped, budgeted, and triaged before the AI ever sees them. The system adapts its pace to your driving style in real time.

Event Detection

49 event types are evaluated every poll cycle (~8 Hz). Each has a configurable threshold, a per-type cooldown, and a priority level. The full threshold catalog is exposed in Settings, with one-tap presets (less / balanced / more aggressive) for quick tuning.

Events that fire but fall within their per-type cooldown are not discarded — they enter a shadow buffer and are still reported to the AI as “things that happened but I stayed quiet about.”

Event Grouping & Budget

Events that occur within a short window (default 300 ms) are grouped into a single “moment” and processed as one compound remark. Three events in quick succession cost one budget slot, not three. Critical events bypass the buffer entirely.

A dynamic budget adapts remark frequency to driving intensity:

Driving Style Budget / Min Cooldown Floor
Calm 4 / min 12s
Normal 8 / min 8s
Spirited 12 / min 3s
Burst 16 / min 2s

AI Awareness Context

Every prompt includes a full awareness context — not just the triggering event, but everything that happened since the last remark:

Event digest — timestamped list of all dispatched events since the last remark.
Suppressed events — events absorbed by cooldowns or blocked by budget, with reasons.
Metric deltas — notable changes in speed, RPM, coolant, fuel, altitude, gear, and load since last time.
Event velocity — events per minute in a rolling 60-second window.
Session counts — per-type totals enabling callbacks like “that’s your 5th hard brake.”

Drive Phase Classification

The AI knows what phase of the drive you’re in. Nine phases are tracked in real time:

Cold Start (first ~90s, near zero speed) → Departure (first ~3 min, building speed) → City (15–50 mph, varied throttle) → Highway (>55 mph, steady) → Spirited (high spirited index) → Traffic (low avg speed, oscillations) → Cruising (steady, minimal input) → Arriving (decelerating toward zero) → Parked (stationary, engine running).

Read-Only OBD Safety

Carsona never writes to your car’s ECU. It uses OBD Mode 01 for live data and Modes 03/07/0A for reading diagnostic codes. Mode 04 (clear DTCs) is permanently prohibited. No UDS commands, no actuator tests, no ECU modifications. Voice commands are strictly read-only through an allowlist, not a blocklist.

Privacy First

Carsona is local-first. Your vehicle data, trip logs, and preferences stay on your device. Optional cloud features (AI, voice) use your own API keys and connect directly — Carsona has no backend.
